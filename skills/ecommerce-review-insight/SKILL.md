# 电商用户评论洞察分析

## 使用场景

当用户已经拿到目标产品的真实用户评价数据，并希望从评论中提炼可执行的商品、详情页、内容、客服、投放或品牌优化建议时，使用本 skill。

本 skill 的目标不是泛泛总结评论，而是把消费者真实反馈整理成 AI 可以分析的结构化素材，再基于真实评论识别：

- 用户主要在讨论什么
- 哪些反馈是正面，哪些反馈是负面
- 谁在买，给谁买
- 用户为什么买，在什么场景下用
- 用户不满意在哪里
- 用户拿哪些竞品或替代品做比较
- 后续详情页、内容种草、客服话术、产品优化和投放人群应该重点关注什么

## 触发条件

用户需求涉及以下任一情况时触发：

- 提供商品评论、买家秀、追评、评分、规格、购买时间等数据，希望做评论分析
- 希望把电商评论整理成 Excel、CSV 或结构化表格
- 希望统计评论类别、情感倾向、负面关键词、满意因素或竞品提及
- 希望从评论中验证购买人群、购买对象和使用场景
- 希望基于真实评论优化详情页卖点、内容种草选题、客服话术或产品体验
- 希望生成购买人群分布图、需求场景分布图、词云图或评论洞察报告

## 核心原则

- 必须以用户提供的真实评论为依据，不编造评论中没有出现的信息。
- 不把产品品类常识直接当作评论结论；只有评论原文明确表达或可以直接判断时，才进行归类。
- 每个重要洞察都尽量配代表性评论原话，证明结论来自真实反馈。
- 无法判断的人群、场景、情感、竞品或原因，统一归入“其他”或“无法判断”。
- 对样本量不足、字段缺失、评论质量较低或评论来源单一的情况，要明确标注分析局限。
- 不编造平台政策、GMV、转化率、行业 benchmark、投放效果、客户案例或未核验的竞品数据。
- 如需使用最新平台规则、公开榜单、价格、店铺数据、投放机制或电商合规要求，先联网核验，并在输出中标注来源和时间。
- 如果用户只提供少量评论，可以先做方向性洞察，但不要包装成统计结论。
- 如果用户要求生成 Excel、图表或词云，优先实际生成文件；当前环境无法生成时，输出可复制的数据表、图表数据和生成说明。

## 必要输入

开始前，先判断用户是否已经提供评论数据。

最少需要：

```text
【评论内容】：用户评价原文
```

建议同时提供：

```text
【评分】：星级、分数或好评/中评/差评
【评论时间】：用户评论或购买时间
【规格】：规格、口味、颜色、尺码、套餐等
【追评】：追加评价内容
【评论人】：昵称、匿名标识或用户 ID
【图片/视频】：是否有图、是否有视频、图片/视频说明
【平台/店铺】：天猫、淘宝、京东、抖音、小红书、拼多多等
【产品名】：目标产品名称
【原先设定的人群和场景】：如已有详情页、人群假设或投放人群，可用于后续验证
【已知竞品名单】：可选，用于降低竞品误判
```

用户可以提供 Excel、CSV、表格截图、复制粘贴的评论文本，或影刀、八爪鱼等工具导出的评论数据。

## 不足信息时的处理方式

- 如果没有评论内容，先要求用户提供评论数据，不要直接分析。
- 如果只有评论内容，没有评分、时间、规格等字段，可以继续做文本分析，并标注“字段缺失”。
- 如果评论数量较少，先输出“方向性分析”，不要给出过度精确的比例判断。
- 如果评论里无法判断购买人群、购买对象或场景，归入“其他”，不要凭品类自行推断。
- 如果评论里没有明确竞品名称或替代品表达，不要强行生成 Top 竞品。
- 如果用户没有提供产品名，先用“目标产品”代称。
- 如果用户没有指定输出文件名，按“产品名 + 评论洞察分析”命名；没有产品名时，使用“商品评论洞察分析”。

## 触发后的开始语

如果用户还没有提供评论数据，使用以下话术：

```text
请先提供目标产品的真实用户评论数据。可以直接粘贴评论文本，也可以上传 Excel / CSV，或提供影刀、八爪鱼等工具导出的评论表。

建议字段包括：评论内容、评分、规格、购买时间、追评、评论人、图片/视频信息。

我会先检查数据字段和样本质量，再按“评论分类 - 情感统计 - 人群场景 - 负面痛点 - 竞品提及 - 词云图 - 经营建议”的顺序生成分析结果。
```

如果用户已经提供评论数据，直接进入阶段 1。

## 工作流程

按以下顺序推进：

1. 数据准备与字段检查
2. 评论清洗与结构化
3. 基础分类与情感统计
4. 购买人群与需求场景分析
5. 负面评论与机会点分析
6. 竞品提及与差异化分析
7. 高频关键词与词云呈现
8. 最终经营建议

不要一开始就直接总结观点。必须先把评论转成可分析的数据结构，再进入洞察判断。

## 阶段 1：数据准备与字段检查

目标：确认用户提供的数据是否足够支撑后续分析。

执行步骤：

1. 读取用户提供的评论数据。
2. 识别已有字段：评论内容、评分、规格、评论时间、购买时间、追评、评论人、图片/视频、平台、店铺、产品名。
3. 判断字段完整度和样本量。
4. 标注数据风险，例如重复评论、空评论、无意义短评、字段缺失、样本量过少、评论来源不明。

输出格式：

```text
数据可用性诊断：
- 样本量：
- 已识别字段：
- 缺失字段：
- 可直接分析的内容：
- 分析局限：
- 是否建议先补充数据：
```

如果样本量和字段足够，继续阶段 2。如果评论内容缺失，暂停并要求用户补充。

## 阶段 2：评论清洗与结构化

目标：把原始评论整理成可统计、可筛选、可追溯的结构化数据。

执行步骤：

1. 去除空评论、明显重复评论和无分析价值的短句。
2. 保留原始评论，不改写用户原文。
3. 将初评和追评作为独立字段；如果需要合并分析，应保留原文可追溯。
4. 统一时间、评分、规格、评论人等字段格式。
5. 为每条评论生成唯一序号，便于后续引用代表性原话。

标准字段：

```text
评论序号｜评论内容｜追评内容｜评分｜规格｜评论时间｜评论人｜图片/视频｜平台/店铺
```

如果生成 Excel 文件，第一张工作表命名为：

```text
原始评论清洗表
```

## 阶段 3：基础分类与情感统计

目标：先把零散评论按固定维度分类，再统计每类评论数量和情感倾向。

评论类别固定为：

- 质量
- 价格
- 包装
- 使用体验
- 售后服务
- 其他

执行要求：

1. 对每条评论判断主要评论类别。
2. 判断情感倾向：正面、负面、中性、无法判断。
3. 如果一条评论同时提到多个维度，优先选择表达最强或最影响购买决策的主类别；必要时可增加“次要类别”字段。
4. 不要把没有明确情绪的陈述强行判为正面或负面。
5. 统计每一类评论的数量、占比和主要情感倾向。

结构化明细表字段：

```text
评论序号｜评论内容｜评论类别｜情感倾向｜评论时间｜评论人｜规格｜评分
```

分类统计表字段：

```text
评论类别｜评论数量｜占比｜正面数量｜负面数量｜中性数量｜主要情感倾向｜代表性原话
```

如果用户要求生成 Excel，文件名默认为：

```text
商品评论分类统计汇总.xlsx
```

如有明确产品名，文件名使用：

```text
产品名-商品评论分类统计汇总.xlsx
```

## 阶段 4：购买人群与需求场景分析

目标：基于阶段 3 已生成的结构化评论表，识别真实评论中出现的购买人群、购买对象和需求场景，验证产品到底被谁买、给谁用、在哪些场景下被使用。

执行要求：

1. 从每条评论中识别用户身份和购买对象。
   - 可识别时，归类为明确人群，例如：自用人群、宝妈/宝爸、子女给父母购买、宠物主人、送礼人群等。
   - 无法明确判断时，统一归入“其他”。
   - 不要根据产品品类自行推断评论中没有表达的人群身份。
2. 从每条评论中提炼主要需求场景。
   - 只提取评论中明确出现或可以直接从原文判断的场景。
   - 无法明确判断时，统一归入“其他”。
   - 不要为了凑场景而扩写用户未表达的信息。
3. 生成两张可视化图表。
   - 第一张图：购买人群分布图，使用饼状图展示不同购买人群的评论数量和占比。
   - 第二张图：需求场景分布图，使用饼状图展示不同需求场景的出现频次和占比。

图表要求：

- 所有图表均需包含清晰标题。
- 各类别需标注评论数量和占比。
- 无法明确判断的评论统一归入“其他”。
- 图表结论必须基于真实评论内容，不要过度推断。

明细表字段：

```text
评论序号｜评论内容｜购买人群｜购买对象｜需求场景｜判断依据原文｜是否明确表达
```

统计表字段：

```text
类别类型｜类别名称｜评论数量｜占比｜代表性原话
```

输出总结：

```text
1. 评论中最主要的购买人群是谁？
2. 用户主要是给谁买？
3. 最高频的使用场景是什么？
```

如果用户提供了原先设定的人群和场景，再补充：

```text
原先设定是否被评论验证：
- 已被验证的人群/场景：
- 评论中出现但原先未重视的人群/场景：
- 评论中证据不足的人群/场景：
```

## 阶段 5：负面评论与机会点分析

目标：聚焦负面评论，找到产品优化、详情页优化、客服话术优化和内容种草机会。

执行步骤：

1. 从负面和中性偏负面的评论中提取高频负面关键词。
2. 统计每个负面关键词的出现频次。
3. 为每个高频负面关键词匹配代表性评论原话。
4. 将用户抱怨翻译成真实痛点。
5. 基于痛点提出可优化机会点。

同时补充分析：

- 评论中哪些因素最常被提及为“满意”的来源
- 品牌口碑的核心驱动因子
- 消费者购买该产品主要想解决的问题

输出表格：

```text
负面关键词｜出现频次｜典型原句｜用户真实痛点｜可优化机会点
```

机会点分类：

- 产品优化机会
- 包装优化机会
- 价格优化机会
- 使用体验优化机会
- 客服与售后优化机会
- 详情页表达优化机会
- 内容种草优化机会

要求：

- 代表性原话必须来自用户评论，不要改写成营销话术。
- 不要把个别低频抱怨包装成普遍问题。
- 如果某类机会点没有评论证据，标注“评论中证据不足”。

## 阶段 6：竞品提及与差异化分析

目标：识别用户在评论中主动提到的其他品牌、其他产品或替代选择，找到真实心智参照物和差异化切入点。

执行步骤：

1. 从评论中识别其他品牌、其他产品、替代选择或对比表达。
2. 统计竞品或替代品被提及的频次。
3. 提取每个竞品被提及时的代表性评论原话。
4. 判断用户是在什么场景下做对比。
5. 判断用户对比后更认可哪一方，并说明原因。
6. 总结本产品可能的差异化切入点。

对比场景可包括：

- 价格
- 效果
- 口感
- 包装
- 使用体验
- 复购
- 送礼
- 服务
- 规格
- 品牌信任
- 其他

输出表格：

```text
竞品名称｜提及频次｜对比场景｜代表性原话｜用户倾向｜差异化机会点
```

最后总结：

```text
用户最常拿哪些竞品来对比？
用户主要在哪些维度上做对比？
我们在哪些地方被用户认为更有优势？
我们在哪些地方弱于竞品？
后续详情页、卖点表达和内容种草应该强化哪些差异化信息？
```

如果评论中没有明确竞品名称，只输出“未发现明确竞品提及”，并分析是否存在模糊替代关系，例如“比之前买的好”“没有以前那款好”“还是老牌子更稳”。

## 阶段 7：高频关键词与词云呈现

目标：把大量评论中的高频信息转成更直观的视觉结果，帮助快速捕捉用户语言。

可生成的词云类型：

- 高频需求词云
- 高频痛点词云
- 高频满意因素词云
- 高频负面关键词词云
- 高频竞品提及词云

执行要求：

1. 从评论中提取高频关键词。
2. 过滤无实际分析价值的通用词、语气词、重复词和平台模板词。
3. 优先保留与产品需求、使用场景、用户痛点、满意因素、负面反馈、竞品提及相关的关键词。
4. 根据关键词出现频次生成词云图，词频越高，字号越大。
5. 如果无法生成图片，输出词频表和可用于生成词云的数据。

词频表字段：

```text
关键词｜出现频次｜关键词类型｜代表性原话
```

要求：

- 不要把“好评”“不错”“满意”这类泛词作为核心洞察，除非它们与具体对象组合出现，例如“包装好”“口感不错”“售后满意”。
- 词云用于辅助理解，不能替代表格统计和原文证据。

## 阶段 8：最终经营建议

目标：把评论洞察转化成商品、运营、内容、客服、投放和详情页团队可以执行的动作。

输出结构：

```text
一、核心发现
- 用户最常讨论的问题：
- 最强正面反馈：
- 最主要负面痛点：
- 最明确购买人群：
- 最高频需求场景：
- 最值得关注的竞品或替代选择：

二、详情页优化建议
- 首屏应该强化：
- 卖点屏应该补充：
- 需要解释或降低顾虑的问题：
- 不建议继续放大的表达：

三、内容种草建议
- 小红书/短视频可优先做的选题：
- 适合使用的用户原话角度：
- 可以放大的真实场景：
- 需要避免的夸大表达：

四、客服话术建议
- 高频售前顾虑：
- 高频售后问题：
- 建议准备的话术方向：

五、产品与包装优化建议
- 产品层面：
- 包装层面：
- 规格/价格层面：
- 使用体验层面：

六、投放与人群建议
- 可优先测试的人群：
- 可优先测试的场景：
- 评论证据：
- 风险或样本局限：
```

所有建议必须回到评论证据。没有评论证据支撑的建议，应标注为“基于经验的补充建议”，不要混同为评论结论。

## 输出文件建议

如果环境支持生成文件，优先输出一个分析包：

```text
产品名-商品评论洞察分析/
├── 产品名-商品评论分类统计汇总.xlsx
├── 购买人群分布图.png
├── 需求场景分布图.png
├── 高频关键词词云图.png
├── 负面关键词词云图.png
├── 竞品提及词云图.png
└── 产品名-商品评论洞察报告.md
```

Excel 建议包含以下工作表：

```text
1. 原始评论清洗表
2. 评论分类明细
3. 分类情感统计
4. 购买人群与场景明细
5. 人群与场景统计
6. 负面关键词与机会点
7. 竞品提及与差异化
8. 高频关键词词频表
```

如果用户只要求文本结果，可以不生成文件，但仍应保留表格结构。

## 质量标准

- 分类结果要能追溯到评论原文。
- 统计口径要前后一致，数量和占比能对应。
- 图表标题清晰，类别名称可读，数量和占比完整。
- 负面痛点必须配代表性原话。
- 竞品提及必须区分明确品牌名、明确产品名和模糊替代关系。
- 结论要能服务商品、运营、内容、设计、客服、销售或投放团队，不停留在空泛描述。
- 对不确定内容明确标注，不用虚构数据填补空白。

## 禁止事项
