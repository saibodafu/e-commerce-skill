# 小红书带货图文卡片策划

## 使用场景

当用户希望基于品类、产品、目标人群、卖点、竞品或常见误区，生成小红书产品种草/带货图文时，使用本 skill。

本 skill 负责输出 2-4 张连续图文卡片的内容结构、每页文案、版式建议和 AI 出图提示词。默认采用小红书知识卡片风格：真实、清爽、专业、有收藏感，不做硬广海报和电商详情页长图。

## 触发条件

用户需求涉及以下任一情况时触发：

- 想为小红书生成产品种草、带货、选购、科普、测评、避坑类图文。
- 想基于一个品类、产品或行业，生成 2-4 张小红书连续图文卡片。
- 想做“小红书科普类图文”，例如讲清楚某个品类怎么选、某个成分/参数/机制是什么意思。
- 想做“小红书测评类图文”，例如多款产品横向对比、维度打分、场景适配、选购推荐。
- 想做“小红书避坑类图文”，例如买前避坑、误区纠正、话术识别、错误选择对比。
- 想把产品卖点转成小红书知识卡片、对比卡片、避坑清单或收藏型图文。
- 想生成每一页的小红书卡片文案、页面结构、视觉版式或 AI 出图提示词。
- 用户提到“小红书”“种草图文”“带货图文”“知识卡片”“测评图文”“避坑图文”“科普图文”“3:4 卡片”“封面+拆解页”等关键词。

以下情况不要使用本 skill：

- 用户要做电商详情页长图、商品详情页原型图、详情页 HTML 线框稿，应使用 `ecommerce-content-wireframe`。
- 用户已经有详情页原型图、包装图，希望逐屏做详情页视觉设计，应使用 `ecommerce-detail-page-visual-design`。
- 用户要做直播脚本、短视频脚本、店铺运营分析、投放策略，不属于本 skill。
- 用户只要单张品牌海报、促销海报、主图、详情页，不属于小红书连续图文卡片时，不使用本 skill。

## 核心原则

- 一个页面只讲一个核心信息：一个知识点、一个测评维度或一个避坑标准。
- 先建立判断标准，再自然呈现产品优势，不要一上来硬推自家产品。
- 文案短句化，避免大段解释；标题吸引收藏，底部总结给出判断。
- 整组图保持统一配色、统一版式语言和统一视觉气质。
- 不编造测试数据、检测结果、平台规则、行业榜单、销量、转化率、客户案例或竞品信息。
- 不伪装真人实测经历；没有真实检测或体验证据时，只能表达为“公开信息观察”“参数对比”“选购参考”。
- 不恶意贬低竞品，不使用“全网第一”“唯一最好”“一定有效”“千万别买”“闭眼入”“吊打”等绝对化或刺激性表达。
- 涉及食品、母婴、美妆、健康、安全、功效、医疗、营养、儿童等内容时，表达要克制，避免治疗化、功效承诺和恐慌表达。
- 如果需要最新平台规则、公开榜单、价格、店铺数据、产品参数、投放机制或合规要求，先联网核验，并在输出中标注来源和时间。

## 必要输入

优先从用户已有信息中提取，不要机械要求用户填写完整表格。缺少非关键字段时，可以做轻量合理假设，并在输出中标注“假设”。

启动所需的关键输入：

```text
品类/产品：
目标用户：
图文主题：
核心卖点或用户痛点：
希望传递的判断标准或倾向性结论：
视觉风格补充：
品牌露出需求：弱露出 / 自然带出 / 可不出现
```

如果是测评类，尽量补充：

```text
自家品牌/产品名：
自家产品核心卖点：
希望放大的优势：
对比产品 A/B/C：
测评维度：
用户最容易纠结的选择点：
```

如果是避坑类，尽量补充：

```text
常见坑点：
用户最容易误判的选择点：
常见错误选择 A/B/C：
希望用户记住的购买理由：
```

如果缺少品类/产品、目标用户、图文主题这 3 个关键字段，先追问。其他字段不足时继续推进，并标注假设。

## 工作流程

按以下顺序执行：

1. 提取用户输入，补齐轻量假设。
2. 判断内容模式：科普类、测评类或避坑类。
3. 判断是否必须联网核验；需要时先查公开资料。
4. 设计 2-4 张翻页结构。
5. 输出逐页内容脚本。
6. 输出逐页 AI 出图提示词。

## 阶段 1：内容模式判断

先判断这组图文属于哪种模式，并说明理由：

| 模式 | 适合场景 | 结构重点 |
|---|---|---|
| 科普类 | 用户需要理解品类、成分、参数、机制、选购标准 | 讲清知识点，让用户知道怎么判断 |
| 测评类 | 用户需要在多个产品、价位、类型或方案中做选择 | 同一标准下对比，让优势自然显现 |
| 避坑类 | 用户容易被价格、包装、概念、话术或错误场景误导 | 先指出误区，再给正确判断标准 |

如果用户没有明确指定模式，选择最适合的一类，并输出：

```text
我判断这次更适合用【科普类/测评类/避坑类】，原因是：[一句话说明]。
如果你想换成另一类，也可以指定。
```

## 阶段 2：资料核验与表达边界

根据品类风险决定是否联网核验。

必须核验的情况：

- 涉及产品公开参数、竞品信息、价格、成分、功效、检测、榜单、平台规则。
- 涉及医学、营养、母婴、儿童、健康、安全、法规或合规表达。
- 用户要求“最新”“真实测评”“公开资料”“数据依据”。

核验后输出简短依据：

```text
资料依据：
- [来源名称]，[发布时间或访问时间]：[可用于本次内容的事实]

表达边界：
- 可以表达：[基于资料能说什么]
- 不建议表达：[没有依据或有风险的说法]
```

如果无法核验或资料不足，继续输出方案，但必须标注：

```text
以下内容按“公开信息观察/参数对比/选购参考”处理，不作为真实检测或实测结论。
```

## 阶段 3：翻页结构

默认生成 2-4 张，优先 3 张。除非用户指定页数，不要超过 4 张。

统一结构：

```text
第 1 张：封面总览图
作用：吸引点击、讲清主题、建立阅读预期。

第 2-4 张：拆解页
作用：每张只讲一个知识点、测评维度或坑点。
```

每页必须包含：

```text
页面定位：
主标题：
副标题：
中间主体：
底部总结：
画面布局：
视觉元素：
风险提醒：
```

## 科普类规则

适用于讲清品类知识、成分机制、参数意义、选购判断标准。

科普类结构：

```text
第 1 张：封面总览图
- 主标题：一图看懂 XXX 怎么选 / XXX 选购前先看
- 副标题：说明为什么值得看
- 中间主体：概括 2-3 个核心知识点
- 底部总结：一句整体判断标准

第 2-4 张：知识点拆解图
- 每张一个知识点
- 用信息图、图标、流程线、卡片框解释
- 底部给出判断标准、行动建议或避坑提醒
```

表达要求：

- 准确、易懂、适合大众阅读。
- 不堆砌专业术语。
- 有争议的信息要克制，不绝对化。
- 最终文案要像小红书收藏卡片，不像论文摘要。

## 测评类规则

适用于多个产品、价位、类型、方案之间的选择比较。

先判断测评逻辑：

| 逻辑 | 适合场景 |
|---|---|
| 横向对比型 | 多个同类产品放在一起比较 |
| 场景适配型 | 不同产品对应不同使用场景 |
| 维度打分型 | 成分、价格、使用感、便携性、适用人群等多维判断 |
| 真实体验型 | 用户提供真实体验素材时，按开箱、使用、细节、感受组织 |

测评类结构：

```text
第 1 张：测评封面图
- 主标题：X 款 XXX 测评，谁更适合 XXX？
- 副标题：说明测评对象和用户痛点
- 中间主体：展示多个产品或 A/B/C 卡片
- 底部总结：说明本组测评要解决的问题

第 2-4 张：测评维度拆解图
- 每张一个测评维度
- 用表格、对比卡片、评分条、清单呈现
- 底部给出选择建议或测评结论
```

表达要求：

- 先建立同一套测评标准，再呈现对比。
- 自家产品可以略微突出，但不要过度抢眼。
- 用“更适合”“相对更友好”“综合来看更均衡”等克制表达。
- 没有真实测试数据时，不写“实测结果”，只写“参数对比”或“公开信息观察”。

## 避坑类规则

适用于买前避坑、误区纠正、营销话术识别、错误选择对比。

先判断避坑逻辑：

| 逻辑 | 适合场景 |
|---|---|
| 误区纠正型 | 用户对品类存在常见错误认知 |
| 选购避坑型 | 用户容易被价格、包装、概念卖点误导 |
| 成分参数型 | 需要看成分、参数、材质、规格的品类 |
| 场景错配型 | 产品不是不好，而是不一定适合某个场景 |
| 话术识别型 | 品类概念包装多，需要拆营销话术 |

避坑类结构：

```text
第 1 张：避坑封面图
- 主标题：买 XXX 前，先避开这几个坑
- 副标题：说明用户容易在哪里选错
- 中间主体：展示几个常见坑点或错误选择
- 底部总结：说明本组避坑重点

第 2-4 张：坑点拆解图
- 每张一个坑点
- 先展示常见误判
- 再解释为什么不够准确
- 最后给出正确选择标准
- 底部给出避坑提醒或选择建议
```

表达要求：

- 不制造焦虑，不直接否定竞品。
- 避免“千万别买”“智商税”等刺激词。
- 错误判断和正确判断要一眼看懂。
- 自家产品优势藏在正确标准里自然出现。

## 视觉规则

通用视觉要求：

- 竖版，3:4 比例。
- 小红书知识卡片风格，真实、清爽、专业、有收藏感。
- 留白充足，不要密密麻麻。
- 可使用产品实拍感图片、小图标、小插画、箭头、流程线、卡片框、对比表格、评分条、勾叉符号。
- 不要像品牌海报，不要像电商详情页，不要促销感，不要硬广感。
- 背景干净，建议浅色底。
- 可使用品牌视觉色作为主色，但品牌露出要按用户要求控制。

根据内容目标选择视觉表达：

| 目标 | 优先视觉风格 |
|---|---|
| 解释复杂信息 | 知识卡片风、框架图风、信息图表风 |
| 建立真实信任 | 实测笔记风、生活场景风、细节观察风 |
| 突出颜值审美 | 杂志拼贴风、氛围种草风、Lookbook 风 |
| 快速帮用户做选择 | 对比表格风、清单打勾风、选购地图风 |
| 降低理解门槛 | 手绘说明风、流程拆解风、图标解释风 |
| 突出专业背书 | 参数拆解风、成分机制风、轻科普风 |
| 强化提醒感 | 避坑清单风、警示标签风、红黄绿判断风 |

资料依据：

- 主标题：
- 副标题：

## AI 出图提示词要求

每页提示词都要包含：

```text
画面类型：小红书 3:4 竖版知识卡片
页面主题：
主标题文字：
副标题文字：
中间主体内容：
底部总结文字：
视觉主体：
版式结构：
配色：
风格：
品牌露出：
禁止事项：
```

提示词必须强调：

- 中文文字清晰可读，短句化，不要大段文字。
- 每张只突出一个核心信息。
- 画面真实、清爽、专业，有收藏感。
- 不要促销标签、低价刺激、夸张广告感。
- 不要新增未经用户确认的功效、参数、数据、认证或竞品结论。

## 默认处理方式

- 用户只给一个产品名和方向时，先追问品类/产品、目标用户、图文主题；如果用户要求直接继续，则在“假设”下完成。
- 用户没有给竞品时，测评类可以使用“同类热门产品、平价款、高端款、传统款、新锐品牌款、用户常见备选产品”等抽象对象，但不能伪造具体品牌参数。
- 用户没有给品牌色时，选择与品类心智匹配的浅色系，并说明为“视觉假设”。
- 用户要求直接出图但信息不足时，先输出逐页提示词；如环境支持图像生成，再按提示词生成图片。
